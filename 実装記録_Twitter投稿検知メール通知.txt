1. 最初の依頼
----------------------------------------------------------------

「とあるTwitterの投稿を検知し、自動でメールを送りたい」

確認した要件：
  - 検知条件 ： 特定アカウント＋キーワード
  - 送信先   ： jsbseven170@gmail.com（自分のメール）
  - 実装方式 ： 無料のGitHub Actions


----------------------------------------------------------------
2. 監視対象の確定
----------------------------------------------------------------

提示URL：https://x.com/daikei_org?s=21

  → 監視対象アカウント ： @daikei_org
  → 検知キーワード     ： 予約


----------------------------------------------------------------
3. 実装方式の選択
----------------------------------------------------------------

Twitter APIは有料のため不使用。以下2案を比較した：

  案A：Playwright（X.com直接スクレイピング）
       メリット  ： X.comを直接開くため確実に動く
       デメリット： Xアカウントのログインが必要な場合あり

  案B：Nitter RSS
       メリット  ： 軽量・ログイン不要
       デメリット ： Nitterサーバーが不安定なことがある

  → ユーザーが案Aの「Playwright」を選択


----------------------------------------------------------------
4. 作成・変更したファイル
----------------------------------------------------------------

【ファイル①】scripts/monitor.py
  メインスクリプト。動作の流れ：
    1. Playwrightでヘッドレスのchromiumブラウザを起動
    2. https://x.com/daikei_org を開く
    3. CSSセレクター article[data-testid="tweet"] でツイートを取得
    4. キーワード「予約」が含まれていればGmailでメール送信
    5. 処理済みツイートIDを .seen_tweet_ids.json に保存（重複通知防止）
  ※ X.comがログインを要求する場合に備え、X_USERNAME/X_PASSWORD での
    ログイン処理もオプションとして実装済み

【ファイル②】.github/workflows/monitor.yml
  GitHub Actionsのワークフロー定義。
    - 15分ごと（cron: "*/15 * * * *"）に自動実行
    - Chromiumブラウザをインストールして monitor.py を実行
    - 実行後、.seen_tweet_ids.json の変更を main にコミット・プッシュ

【ファイル③】requirements.txt
  Pythonの依存ライブラリを記載。
    playwright>=1.44.0


----------------------------------------------------------------
5. GitHub Secrets の設定
----------------------------------------------------------------

コードにパスワード等を直接書かないため、GitHub Secretsで管理。
設定場所：https://github.com/woodstockv999/-s/settings/secrets/actions

  シークレット名       設定値
  -------------------------------------------------------
  KEYWORDS             予約
  GMAIL_ADDRESS        jsbseven170@gmail.com
  GMAIL_APP_PASSWORD   Gmailアプリパスワード（16桁）
  X_USERNAME           （任意）Xアカウントのユーザー名
  X_PASSWORD           （任意）Xアカウントのパスワード

【Gmailアプリパスワードの取得手順】
  1. Googleアカウント → セキュリティ → 2段階認証を有効化
  2. 「アプリパスワード」→ アプリ名を任意入力 → 生成
  3. 表示された16桁のパスワードを GMAIL_APP_PASSWORD に設定

----------------------------------------------------------------
7. 技術ポイントまとめ
----------------------------------------------------------------

Playwright
  ChromiumブラウザをPythonコードで操作するライブラリ。
  JavaScriptで動的にレンダリングされるX.comにも対応できる。

GitHub Actions 無料枠
  月2000分の実行時間が無料。15分ごとの監視なら十分に収まる。

Gmailアプリパスワード
  Googleの2段階認証を有効にした上で発行する専用パスワード。
  通常のGmailパスワードをコードに書く必要がなく安全。

重複通知防止
  処理済みのツイートIDを .seen_tweet_ids.json に保存することで、
  同じツイートに対して何度もメールが送られることを防いでいる。
